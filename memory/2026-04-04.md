## Lessons Learned

1. **JavaScript obfuscation ≠ security** — Always deobfuscate client-side code
2. **Authorization checks matter** — IDOR is still prevalent in modern apps
3. **User input sanitization** — Never pass user input to system commands
4. **.env files in web roots** — Classic mistake, always check for exposed configs
5. **Kernel vulnerabilities** — Email hints can point to specific CVEs
6. **Race conditions** — Some exploits require precise timing with multiple shells
7. **CVE-2023-0386** — OverlayFS/FUSE privilege escalation requires:
   - Kernel 5.x (before patch)
   - /dev/fuse available
   - Two concurrent processes (race condition)

## Kubernetes Learning Notes

### What is Kubernetes?
Kubernetes (K8s) is an **orchestration platform** for managing containerized applications at scale. Think of it as a "datacenter operating system" that:
- Runs containers across multiple machines (nodes)
- Handles scaling, load balancing, and self-healing
- Manages networking between containers
- Provides storage and secrets management

### What is a Kubelet?
The **kubelet** is an agent that runs on every node in a Kubernetes cluster. Its job is to:
- Communicate with the control plane (kube-apiserver)
- Start/stop containers based on Pod specifications
- Report node and container health/status
- Expose node-level metrics

**The vulnerability here:** The kubelet's API (port 10250) is **unauthenticated** on this box, meaning anyone can:
- List all pods running on the node
- Read container logs
- Execute commands inside containers
- Access the host filesystem via volume mounts

### Why This Matters for Pentesting
- **Kubelet API (10250)** should NEVER be exposed to untrusted networks
- Unauthenticated kubelet = full container control = potential host escape
- Volume mounts can expose host filesystem into containers
- Container logs may contain sensitive data
