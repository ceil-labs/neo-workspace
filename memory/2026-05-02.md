### Consolidated Learnings ÔÇö AD Internal Pentest Playbook

#### 1. crackmapexec Ôëá smbmap ÔÇö Always Cross-Verify
- crackmapexec against DC01 with hostname returned **empty**
- smbmap revealed `Department Shares` with full folder structure
- **Rule:** When shares seem sparse, always run smbmap as secondary check
- crackmapexec works better with **IPs** than hostnames for consistent output

#### 2. Pass-the-Hash Protocol Matrix
| Protocol | PtH Works? | Notes |
|----------|-----------|-------|
| SMB (crackmapexec) | ÔŁî Sometimes fails | STATUS_LOGON_FAILURE possible |
| WinRM (evil-winRM) | Ôťů More reliable | Same hash that SMB rejected |
| RDP (xfreerdp) | ÔŁî Often blocked | "Account restrictions" = NTLMv2-only policy |
| **Lesson:** If PtH fails on one protocol, try others immediately |

#### 3. SQL Server = SYSTEM via SeImpersonatePrivilege
- SQL service accounts (`NT SERVICE\MSSQL$SQLEXPRESS`) often have `SeImpersonatePrivilege`
- Check with: `whoami /all` from `xp_cmdshell`
- **PrintSpoofer** reliably escalates: `PrintSpoofer64.exe -i -c <command>`
- Upload payload via SQL `upload` command (base64-encoded, auto-decoded)
- msfvenom reverse shell + PrintSpoofer = interactive SYSTEM shell

#### 4. Inveigh in Constrained Shells (evil-winRM)
- evil-winRM PTY cannot handle `Console.KeyAvailable` (interactive keypress loop)
- **Fix 1:** Use `-RunTime <minutes>` for auto-exit without keypress
- **Fix 2:** Use `Start-Job` with `-ConsoleOutput N` (background execution)
- Captured CT059 hash from DC01 SMB authentication attempts

#### 5. bloodhound-python Hash Syntax
- `--hashes` requires `LMHASH:NTHASH` with colon separator
- Empty LM hash: `aad3b435b51404eeaad3b435b51404ee`
- Full: `--hashes 'aad3b435b51404eeaad3b435b51404ee:NTLMHASH'`

#### 6. Domain Admin Group Membership ÔÇö Fresh Auth Required
- Added CT059 to Domain Admins successfully
- `type \\DC01\C$\...` from existing session = Access Denied
- **Kerberos tickets don't update in existing sessions**
- **Fix:** Use new authentication (crackmapexec, new RDP, new WinRM) for fresh ticket
